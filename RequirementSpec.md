# BLEフットペダル型キーボードデバイス 要件書

## 1. 概要

譜めくり用途で使用する、フットペダル操作によりBluetooth Low Energy (BLE) 経由でiPadに矢印キー(→)を送信するワイヤレスキーボードデバイスを開発する。

## 2. 目的

演奏中・作業中に手を使わずペダル操作のみでiPad上の譜面や資料を次ページに送れるようにする。

## 3. 用語定義

| 用語 | 説明 |
|---|---|
| ペダル | 単純なON/OFFスイッチ(フットスイッチ) |
| HID | Human Interface Device。BLE経由でキーボード入力として認識される仕組み(HOG: HID over GATT) |
| クールダウン | 誤操作による連続送信を防ぐための不応期間 |

## 4. 機能要件

### 4.1 基本動作

| ID | 要件 |
|---|---|
| F-01 | ペダルを踏んで離した時点(リリースエッジ)で、キーボードの右矢印キー(→)を1回送信する |
| F-02 | 踏みっぱなしにしても、離すまでは追加のキー送信を行わない |
| F-03 | キー送信後、10秒間は再送信を行わない(クールダウン期間中のペダル操作は無視する) |
| F-04 | クールダウン期間中の操作は無視するのみとし、キューイングや遅延実行は行わない |

### 4.2 接続・ペアリング

| ID | 要件 |
|---|---|
| F-05 | iPadとBLE HID(HOG)キーボードとしてペアリング・接続できること |
| F-06 | 一度ペアリングした後は、iPad側の「設定」操作なしに自動再接続(ボンディング)できること |
| F-07 | 接続確立後は、ペダル操作から矢印キー送信までの遅延を体感できないレベル(即応)とする |

### 4.3 省電力・スリープ

| ID | 要件 |
|---|---|
| F-08 | ペダル操作がない状態が30分間継続した場合、自動的にBLE接続を切断し低消費電力状態(スリープ)に移行する |
| F-09 | スリープ中にペダルが踏まれた場合、デバイスが復帰しBLEの再アドバタイズ/再接続を開始する |
| F-10 | スリープからの復帰〜再接続までの遅延は1〜2秒程度まで許容する |
| F-11 | 復帰後、再接続が完了し通常動作状態になった以降は、F-07の即応性要件を満たすこと |

### 4.4 バッテリー監視・報告

| ID | 要件 |
|---|---|
| F-12 | 約5秒間隔でバッテリー電圧を測定し、シリアルコンソールへログ出力すること(開発・デバッグ用) |
| F-13 | F-12で測定した電圧を残量%に変換し、標準BLE Battery Service (0x180F) 経由でiPadに報告すること。iPad側の「設定」>「Bluetooth」やバッテリーウィジェットでの表示はiOS標準機能に依存し、本デバイスはBattery Level characteristic (0x2A19) への書き込みまでを担当する |

## 5. 非機能要件

### 5.1 電源

| ID | 要件 |
|---|---|
| N-01 | 充電式バッテリー(リチウムポリマー等)で駆動すること |
| N-02 | 1回の満充電で、通常使用(演奏中の利用を想定した間欠的なペダル操作+休憩中のスリープ移行)で最低1日は動作すること |
| N-03 | USB等での充電に対応すること |
| N-08 | 充電電流は100mAとする(XIAO内蔵充電ICのデフォルト50mAから高速充電レートに変更) |

### 5.2 応答性

| ID | 要件 |
|---|---|
| N-04 | 接続維持中のキー送信遅延は無視できるレベル(数十ms以下を目安)であること |
| N-05 | 初回起動時、およびスリープ復帰時の接続確立に要する時間は1〜2秒以内を目安とする |

### 5.3 信頼性

| ID | 要件 |
|---|---|
| N-06 | チャタリング(スイッチの接点バウンド)による誤検知・誤送信を防止すること |
| N-07 | 演奏中の連続使用において、キー送信の取りこぼしがないこと |

## 6. ハードウェア構成

| 項目 | 内容 |
|---|---|
| マイコンボード | Seeed Studio XIAO nRF52840(Nordic nRF52840、BLE 5.0 内蔵) |
| 入力デバイス | フットペダル(単純な機械式ON/OFFスイッチ。購入した実機は常閉接点(NC、踏むと開く)) |
| 接続方式 | GPIO 1ピン + GND、内部プルアップ抵抗を使用 |
| 電源 | 充電式LiPoバッテリー + XIAO内蔵充電回路(USB-C充電対応、充電電流100mA) |

## 7. ソフトウェア構成

| 項目 | 内容 |
|---|---|
| ファームウェア | CircuitPython |
| 使用ライブラリ | `adafruit_ble`、`adafruit_hid` |
| 組み込みモジュール | `keypad`(ペダル入力のデバウンス・エッジ検出、追加ライブラリ不要) |
| 通信方式 | BLE HID over GATT(HOG)、Appearance: Generic Keyboard |
| 送信キーコード | Keycode.RIGHT_ARROW |
| バッテリー報告 | 標準BLE Battery Service (0x180F)。電圧→残量%は区分線形の近似カーブ(データシート値ではなく一般的な目安) |

## 8. 制約事項・前提条件

- 接続先はiPad(iPadOS)を主対象とする。他のOS(Mac/Windows/Android等)での動作は本要件の対象外とする
- ペダルは単純なメカニカルスイッチを想定し、静電容量式や光学式等の特殊なスイッチは対象外とする
- 日本語入力(IME操作等)は対象外。あくまで矢印キー1種類の送信のみを扱う
- ペアリング時の初回設定(iPad側のBluetooth設定画面での選択操作)はユーザー操作を要する

## 9. 除外事項(スコープ外)

- 複数キーの組み合わせ送信
- 複数デバイス(複数iPad等)への同時接続・切り替え
- スマホアプリ等によるキー割り当てのカスタマイズ機能
- 充電中かどうかの検出・報告(充電状態を示す`CHARGE_STATUS`ピンは存在するが未使用。F-13はあくまで残量%のみ)

## 10. 今後の検討事項

- 実機でのバッテリー持続時間の実測(F-12のバッテリー電圧ログを活用)、および必要に応じたスリープ移行時間(30分)の調整
- F-13の電圧→残量%カーブは一般的な目安値であり、実機のバッテリーで実測して調整する余地がある
- チャタリング対策のデバウンス時間の実機調整
- 長期使用時の安定性検証(ボンディング情報の永続性、再接続の安定性)
- 筐体・防水性等、演奏環境での物理的な使用条件の検討
- 常開接点(NO)のペダルに交換する場合、現状は`keypad.Keys`と`PinAlarm`のプル方向設定がNC専用にハードコードされているため作り直しが必要(詳細はCLAUDE.mdの「PEDAL_PRESSEDの注意点」参照)
